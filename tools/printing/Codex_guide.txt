PDF AUTO-PRINT SYSTEM - CODEX GUIDE
=====================================

OVERVIEW
--------
This is a PDF Auto-Print & Archive System with Adobe Illustrator integration for macOS. The goal is to automatically print PDFs with correct scaling (30cm x 39cm output) using the SHARP MX-6580N printer.

CRITICAL PROBLEM WE'RE SOLVING
------------------------------
- PDFs print at 29.9cm x 38.9cm instead of target 30cm x 39cm (missing 1mm in each dimension)
- Need to achieve 100.2% scaling to get correct size
- SHARP printer ignores command-line scaling parameters
- GUI automation causes rotation and positioning issues

CURRENT WORKING SOLUTION
------------------------
1. Script opens PDF in Preview.app
2. Opens print dialog (Cmd+P)
3. User manually verifies/adjusts these CRITICAL settings:
   - Preset: ADHESIVE
   - Paper Size: ADHESIVE (330 x 480 mm)
   - Pages: Range from 1 to 1
   - Preview → Auto Rotate: OFF
   - Preview → Scale: 100.2% (MOST IMPORTANT - often shows as 101% and needs manual correction)
4. User clicks Print manually
5. Script archives file automatically

WHAT DOESN'T WORK - AVOID THESE APPROACHES
------------------------------------------
❌ Command-line scaling with lp command (-o scaling=X) - SHARP printer ignores this completely
❌ Custom page sizes larger than paper dimensions - causes content to be cut off
❌ Margin controls (-o page-left=0, etc.) - no effect on actual print size
❌ Python PDF preprocessing with PyPDF2/Quartz - causes rotation issues
❌ Ghostscript PDF resizing - print jobs fail to send
❌ Complex GUI automation to set print dialog values - unreliable and causes syntax errors
❌ fit-to-page option - distorts aspect ratio and causes wrong orientation
❌ Preview.app automated printing via System Events - causes rotation problems

TECHNICAL FINDINGS
------------------
- SHARP MX-6580N does NOT support scaling parameter in lp command
- Available printer options (via lpoptions -l) show no scaling capability
- ADHESIVO preset saves paper size, page range, input slot but NOT scale percentage
- Scale setting defaults to 101% but needs to be exactly 100.2% for correct output
- GUI print dialog scaling works, command-line scaling does not
- Printer has two configurations: PostScript driver and AirPrint driver

- previewAndConfirm() - opens PDF in Preview for user confirmation
